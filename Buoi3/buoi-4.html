Class, đối tượng & đóng gói

Mô hình hóa thế giới bằng thuộc tính và hành vi.

Mục tiêu

  • Phân biệt class và object
  • Tổ chức code bằng package
  • Áp dụng đóng gói
  • Dùng constructor, getter, setter

1. Tư duy OOP

OOP tổ chức chương trình quanh các đối tượng. Mỗi đối tượng có trạng thái (thuộc tính) và hành vi (phương thức). Ví dụ SinhVien có mã, tên, điểm và hành vi học, tính xếp loại.

Bốn tính chất chính: đóng gói, kế thừa, đa hình, trừu tượng. Buổi này tập trung vào đóng gói.

2. Package, class và object

package buoi4;

public class Student {
    private String id;
    private String name;
    private double score;

    public String getRank() {
        return score >= 8 ? "Giỏi" : score >= 5 ? "Đạt" : "Chưa đạt";
    }
}

Class là bản thiết kế; object là thực thể được tạo từ class. Một class có thể tạo nhiều object độc lập.

4. Constructor, getter, setter

public Student(String id, String name, double score) {
    this.id = id;
    this.name = name;
    setScore(score);
}

public String getName() { return name; }

public void setName(String name) {
    if (name == null || name.isBlank())
        throw new IllegalArgumentException("Tên không được trống");
    this.name = name.trim();
}

public void setScore(double score) {
    if (score < 0 || score > 10)
        throw new IllegalArgumentException("Điểm phải từ 0 đến 10");
    this.score = score;
}

Constructor trùng tên class, không có kiểu trả về và chạy khi dùng new. Getter đọc dữ liệu; setter cập nhật và kiểm tra tính hợp lệ.

5. Thực hành & bài tập

Quản lý sản phẩm: Class Product gồm id, name, price, quantity; constructor; getter/setter; phương thức tính thành tiền và in thông tin.
  1. Tạo class Rectangle, kiểm tra cạnh dương, tính chu vi/diện tích.
  2. Tạo class BankAccount với deposit, withdraw; không cho số dư âm.
  3. Tạo package model chứa class và app chứa Main.
