1. OOP trong Java

Khái niệm: OOP (Object‑Oriented Programming – Lập trình hướng đối tượng) là phương pháp tổ chức chương trình quanh các đối tượng (object). Mỗi đối tượng bao gồm trạng thái (thuộc tính / field) và hành vi (phương thức / method). Thay vì viết code tuần tự, OOP cho phép mô hình hóa thế giới thực thành các class và object tương tác với nhau.

Ví dụ: Lớp Student có các thuộc tính id, name, score và phương thức getRank() để xác định xếp loại. Mỗi sinh viên cụ thể là một object được tạo từ class này.

Bốn tính chất chính của OOP:

  • Đóng gói (Encapsulation) – Che giấu dữ liệu bên trong class, chỉ cho phép truy cập qua phương thức công khai (getter/setter). Giúp bảo vệ dữ liệu khỏi bị thay đổi trái phép.
  • Kế thừa (Inheritance) – Một class con có thể thừa hưởng thuộc tính và phương thức từ class cha, giúp tái sử dụng code. Ví dụ: SinhVienIT extends Student.
  • Đa hình (Polymorphism) – Cùng một phương thức nhưng có thể hoạt động khác nhau tùy vào đối tượng gọi nó. Ví dụ: getRank() có thể bị override ở class con để tính điểm khác.
  • Trừu tượng (Abstraction) – Ẩn đi chi tiết triển khai, chỉ hiển thị giao diện cần thiết. Sử dụng abstract class hoặc interface.
// Ví dụ minh họa OOP cơ bản
public class DemoOOP {
    public static void main(String[] args) {
        Student s = new Student("SV01", "An", 8.5);
        System.out.println(s.getRank()); // Giỏi
    }
}
🚨 Chú ý – Lỗi người mới hay mắc:
  • ❌ Quên viết hoa chữ cái đầu tên class: demoOOP → ✅ Phải là DemoOOP (quy tắc PascalCase).
  • ❌ Viết sai chữ ký main: public static void Main(String[] args) → ✅ Phải là main viết thường.
  • ❌ Dùng System.out.Println (viết hoa P) → ✅ Phải là System.out.println (viết thường).
  • ❌ Quên import class Student khi nằm khác package → ✅ Thêm import com.example.school.Student;.
  • ❌ Nhầm lẫn class và object: Class là bản thiết kế, object là thực thể tạo ra bằng new.

2. Package, Class và Object

Khái niệm:
  • Package – Là cách tổ chức các class vào các thư mục logic, giống như thư mục trong hệ thống file. Cú pháp: package ten_package; (đặt ở dòng đầu tiên của file). Quy ước đặt tên: viết thường, dùng dấu chấm phân cách, thường theo tên miền đảo ngược. Ví dụ: com.example.school.
  • Class (Lớp) – Là bản thiết kế (blueprint) mô tả cấu trúc và hành vi của đối tượng. Một file .java chỉ được chứa tối đa một class public, và tên class public phải trùng với tên file.
  • Object (Đối tượng) – Là thực thể cụ thể được tạo ra từ class bằng từ khóa new. Ví dụ: Student s = new Student("SV01", "An", 8.5); tạo một object s từ class Student.
package com.example.school;

public class Student {
    private String id;
    private String name;
    private double score;

    public Student(String id, String name, double score) {
        this.id = id;
        this.name = name;
        this.score = score;
    }

    public String getRank() {
        return score >= 8 ? "Giỏi" : score >= 5 ? "Đạt" : "Chưa đạt";
    }
}
🚨 Chú ý – Lỗi người mới hay mắc:
  • ❌ Đặt tên file không trùng với tên class public: File là student.java nhưng class là Student → ✅ File phải là Student.java.
  • ❌ Quên khai báo package ở dòng đầu hoặc đặt sai vị trí (sau import) → ✅ package luôn là dòng đầu tiên trong file.
  • ❌ Đặt 2 class public trong cùng 1 file → ✅ Mỗi file chỉ được 1 class public.
  • ❌ Cấu trúc thư mục không khớp package: Khai báo package com.example.school nhưng file không nằm trong thư mục com/example/school/ → ✅ Thư mục phải khớp chính xác.
  • ❌ Nhầm lẫn class và object: Student là class (bản thiết kế), new Student(...) tạo ra object (thực thể).

Class là bản thiết kế; object là thực thể được tạo từ class bằng từ khóa new. Một class có thể tạo ra nhiều object khác nhau.

3. Thuộc tính & Phương thức

Khái niệm:
  • Thuộc tính (Attribute / Field) – Là các biến được khai báo bên trong class, dùng để lưu trữ trạng thái của đối tượng. Cú pháp: access_modifier kieu_du_lieu ten_thuoc_tinh;. Ví dụ: private String name;
  • Phương thức (Method) – Là hàm được khai báo bên trong class, dùng để định nghĩa hành vi của đối tượng. Cú pháp: access_modifier kieu_tra_ve ten_phuong_thuc(tham_so) { ... }. Ví dụ: public String getName() { return name; }
  • Từ khóa this – Tham chiếu đến đối tượng hiện tại. Dùng để phân biệt thuộc tính của class và tham số phương thức khi trùng tên. Ví dụ: this.name = name;
Thành phầnCú phápÝ nghĩa
Thuộc tínhprivate String name;Lưu trữ tên sinh viên
Phương thứcpublic String getName() { return name; }Trả về giá trị tên
Phương thức voidpublic void setName(String name) { this.name = name; }Gán giá trị tên (không trả về gì)
Student s = new Student("SV02", "Bình", 7.0);
System.out.println(s.getName()); // Bình
🚨 Chú ý – Lỗi người mới hay mắc:
  • ❌ Truy cập trực tiếp thuộc tính private từ bên ngoài: s.name → ✅ Phải dùng getter: s.getName().
  • ❌ Quên từ khóa return trong phương thức có kiểu trả về: public String getName() { name; } → ✅ Phải có return name;.
  • ❌ Nhầm void và kiểu trả về: Phương thức void không trả về giá trị, nếu viết return name; trong method void sẽ bị lỗi biên dịch.
  • ❌ Quên dùng this khi tham số trùng tên thuộc tính: name = name; (gán chính nó) → ✅ this.name = name;.
  • ❌ Gọi phương thức nhưng quên dấu ngoặc (): s.getName → ✅ s.getName().

4. Phạm vi truy cập (Access Modifiers)

Khái niệm: Access Modifier (phạm vi truy cập) là từ khóa xác định ai được phép truy cập vào thuộc tính, phương thức hoặc class. Java có 4 mức truy cập từ chặt nhất đến rộng nhất: private → default (không ghi gì) → protected → public. Việc chọn đúng modifier là nền tảng của nguyên tắc Đóng gói (Encapsulation).
ModifierCùng classCùng packageClass con (khác package)Mọi nơi
private✓———
default (không ghi gì)✓✓——
protected✓✓✓—
public✓✓✓✓
public class Student {
    private String name;      // Chỉ truy cập được trong class Student
    String school;            // default – cùng package mới truy cập được
    protected double score;   // cùng package + class con
    public String id;         // truy cập được ở mọi nơi
}
🚨 Chú ý – Lỗi người mới hay mắc:
  • ❌ Đặt thuộc tính là public: public String name; → ✅ Nên dùng private + getter/setter để bảo vệ dữ liệu (nguyên tắc Encapsulation).
  • ❌ Nhầm default là từ khóa: Viết default String name; → ✅ Default access nghĩa là không viết gì: String name;
  • ❌ Nghĩ protected chỉ dành cho class con → ✅ protected còn cho phép truy cập từ cùng package.
  • ❌ Dùng private cho class ngoài cùng (top-level class) → ✅ Top-level class chỉ có thể là public hoặc default, không thể private hay protected.
Quy tắc vàng: Thuộc tính nên là private; cung cấp phương thức public getter/setter để kiểm soát truy xuất. Đây chính là nguyên tắc Encapsulation.

5. Constructor, Getter, Setter

Khái niệm:
  • Constructor (Hàm khởi tạo) – Là phương thức đặc biệt được gọi tự động khi tạo object bằng new. Tên constructor phải trùng với tên class và không có kiểu trả về (kể cả void). Dùng để gán giá trị ban đầu cho các thuộc tính.
  • Default Constructor – Nếu bạn không viết constructor nào, Java tự tạo một constructor rỗng (không tham số). Nhưng khi bạn viết ít nhất 1 constructor có tham số, constructor mặc định sẽ biến mất.
  • Getter – Phương thức public dùng để đọc giá trị thuộc tính private. Quy ước: getXxx() hoặc isXxx() (cho boolean).
  • Setter – Phương thức public dùng để ghi / thay đổi giá trị thuộc tính private. Quy ước: setXxx(value). Nên kiểm tra dữ liệu đầu vào (validation) trước khi gán.
// Constructor – khởi tạo đối tượng
public Student(String id, String name, double score) {
    this.id = id;
    this.name = name;
    setScore(score); // Gọi setter để validate ngay trong constructor
}

// Getter – đọc giá trị
public String getName() { return name; }

// Setter – ghi giá trị có kiểm tra
public void setName(String name) {
    if (name == null || name.isBlank())
        throw new IllegalArgumentException("Tên không được trống");
    this.name = name.trim();
}

public void setScore(double score) {
    if (score < 0 || score > 10)
        throw new IllegalArgumentException("Điểm phải từ 0 đến 10");
    this.score = score;
}
🚨 Chú ý – Lỗi người mới hay mắc:
  • ❌ Đặt kiểu trả về cho constructor: void Student(...) → ✅ Constructor không có kiểu trả về: public Student(...).
  • ❌ Tên constructor không trùng tên class: public student(...) → ✅ Phải viết đúng: public Student(...) (PascalCase, trùng tên class).
  • ❌ Quên tạo constructor không tham số khi đã có constructor có tham số: Student s = new Student(); sẽ lỗi → ✅ Thêm constructor rỗng nếu cần: public Student() {}.
  • ❌ Setter không validate dữ liệu: public void setScore(double score) { this.score = score; } → ✅ Luôn kiểm tra: điểm từ 0–10, tên không null/blank.
  • ❌ Getter trả sai kiểu: public int getName() nhưng name là String → ✅ Kiểu trả về phải khớp với kiểu thuộc tính.
  • ❌ Quên this trong constructor/setter khi tham số trùng tên: name = name; → ✅ this.name = name;.

Constructor khởi tạo giá trị ban đầu; getter trả về giá trị; setter kiểm tra và gán giá trị. Ba thành phần này tạo nên nền tảng của nguyên tắc Đóng gói (Encapsulation).

// Ví dụ sử dụng constructor, getter, setter
Student s = new Student("SV03", "Cường", 9.0);
System.out.println(s.getRank()); // Giỏi
s.setScore(6.5);
System.out.println(s.getRank()); // Đạt

// Thử gán giá trị không hợp lệ → sẽ ném exception
// s.setScore(15); // IllegalArgumentException: Điểm phải từ 0 đến 10
