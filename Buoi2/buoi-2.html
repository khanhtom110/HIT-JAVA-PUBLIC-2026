1. Biến và kiểu dữ liệu

Cú pháp khai báo biến: kieu_du_lieu ten_bien = gia_tri;

Trong Java, kiểu dữ liệu được chia làm 2 loại chính: Primitive (Nguyên thủy) và Reference (Tham chiếu).

1.1. Các kiểu dữ liệu cơ bản

Phân loại Kiểu dữ liệu Kích thước Giá trị mặc định Ví dụ trong Java Ý nghĩa
Primitive
(Nguyên thủy)		 int 4 bytes 0 25 Số nguyên thông thường
long 8 bytes 0L 3_000_000_000L Số nguyên lớn (phải thêm hậu tố L hoặc l)
double 8 bytes 0.0 8.5 Số thực (mặc định của số thực trong Java)
boolean 1 bit false true / false Logic Đúng/Sai
char 2 bytes '\u0000' 'A', '9' Một ký tự duy nhất (nằm trong nháy đơn ' ')
Reference
(Tham chiếu)		 String Phụ thuộc chuỗi null "An", "Java Public" Chuỗi ký tự văn bản (nằm trong nháy kép " ")
🚨 3 Điểm "Bẫy" cần lưu ý:
  • Hậu tố L của kiểu long: Nếu viết long n = 3000000000; sẽ bị lỗi compile vì Java mặc định hiểu các số nguyên ghi ra là kiểu int (mà số này vượt quá giới hạn int). Bắt buộc phải thêm chữ L thành 3_000_000_000L.
  • Nháy đơn vs Nháy kép: Kiểu char bắt buộc dùng nháy đơn ('A'), còn String bắt buộc dùng nháy kép ("An"). Viết lộn xộn code không biên dịch được.
  • Chữ hoa chữ thường: Các kiểu nguyên thủy viết thường hoàn toàn (int, long...), riêng String là một Class (kiểu tham chiếu) nên viết hoa chữ S theo đúng chuẩn PascalCase.

1.2. Ép kiểu dữ liệu (Type Casting)

Ép kiểu là quá trình chuyển đổi một biến từ kiểu dữ liệu này sang kiểu dữ liệu khác. Trong Java có 2 loại ép kiểu:

// 1. Ép kiểu tự động (Widening / Implicit Casting)
int myInt = 9;
double myDouble = myInt; // Tự động chuyển từ int sang double
System.out.println(myDouble); // Kết quả: 9.0

// 2. Ép kiểu thủ công (Narrowing / Explicit Casting)
double pi = 3.14;
int roundedPi = (int) pi; // Bắt buộc phải ghi rõ kiểu dữ liệu trong ngoặc (...)
System.out.println(roundedPi); // Kết quả: 3 (Phần thập phân .14 đã bị cắt bỏ)
Loại ép kiểu Cơ chế An toàn dữ liệu Chiều chuyển đổi
Tự động
(Widening)		 Java tự xử lý khi chuyển từ kiểu có **kích thước nhỏ** sang kiểu có **kích thước lớn hơn**. ✅ An toàn tuyệt đối, không sợ mất dữ liệu. byte → short → char → int → long → float → double
Thủ công
(Narrowing)		 Lập trình viên phải tự ép bằng cú pháp (kiểu_dữ_liệu) khi chuyển từ **lớn** về **nhỏ**. ❌ Rủi ro, có thể gây mất hoặc tràn dữ liệu (như mất phần thập phân, tràn số). double → float → long → int → char → short → byte
💡 Bài toán chia số nguyên kinh điển (Bẫy phỏng vấn/kiểm tra):
Nếu viết double result = 7 / 2;, học viên thường nghĩ kết quả là 3.5.
Nhưng thực tế: Java tính 7 / 2 (int / int) bằng 3 trước, rồi mới gán 3 đó vào biến double thành 3.0.
👉 Cách xử lý đúng: Phải ép kiểu cho ít nhất một vế: double result = (double) 7 / 2; hoặc viết thẳng 7.0 / 2 để ra kết quả chuẩn 3.5.

2. Nhập và xuất dữ liệu (Input & Output)

Để giao tiếp với người dùng, Java cung cấp các phương thức xuất dữ liệu chuẩn qua System.out và đọc dữ liệu nhập vào thông qua lớp Scanner.

2.1. Xuất dữ liệu (Output)

Java có 3 phương thức xuất dữ liệu ra màn hình Console phổ biến:

  • System.out.print(): In nội dung ra màn hình nhưng không xuống dòng.
  • System.out.println(): In nội dung xong và tự động xuống dòng tiếp theo.
  • System.out.printf(): In dữ liệu có định dạng (Format).
    💡 Định dạng hay dùng: %s (Chuỗi), %d (Số nguyên), %.2f (Số thực lấy 2 chữ số thập phân), %n (Xuống dòng chuẩn hệ điều hành).

2.2. Nhập dữ liệu (Input với Scanner)

Để nhập dữ liệu từ bàn phím, ta cần import thư viện java.util.Scanner và khởi tạo đối tượng Scanner.

import java.util.Scanner;

public class DemoInput {
    public static void main(String[] args) {
        // Khởi tạo công cụ quét dữ liệu đầu vào chuẩn (bàn phím)
        Scanner scanner = new Scanner(System.in);

        // 1. Đọc một chuỗi ký tự (Họ tên)
        System.out.print("Nhập họ tên: ");
        String name = scanner.nextLine();

        // 2. Đọc một số nguyên bằng cách đọc chuỗi rồi chuyển kiểu (Khuyên dùng)
        System.out.print("Nhập tuổi: ");
        int age = Integer.parseInt(scanner.nextLine());

        // In kết quả ra màn hình bằng printf
        System.out.printf("Xin chào %s - Bạn %d tuổi.%n", name, age);
    }
}
🚨 Giải mã cơ chế: Tại sao nên dùng nextLine() rồi ép kiểu thay vì dùng nextInt()?

Khi bạn gõ số 18 rồi nhấn Enter, trong bộ đệm bàn phím sẽ lưu chuỗi ký tự là "18\n" (với \n là ký tự xuống dòng khi ấn Enter).

  • Nếu dùng nextInt(): Nó chỉ hốt duy nhất số 18 đi, và để lại ký tự thừa \n chơ vơ trong bộ đệm. Khi có lệnh nextLine() ngay phía sau, lệnh này nhìn thấy \n thì lập tức đớp luôn và nghĩ rằng người dùng nhập chuỗi rỗng. Kết quả là chương trình bị trôi (bỏ qua dòng).
  • Nếu dùng nextLine(): Nó sẽ hốt sạch cả cụm "18\n", dọn dẹp bộ đệm hoàn toàn sạch sẽ. Sau đó hàm Integer.parseInt() chỉ việc bóc tách chuỗi "18" thành số 18. Cực kỳ an toàn và sạch code!

3. Toán tử trong Java (Operators)

Toán tử dùng để thực hiện các phép toán trên biến và giá trị. Java chia toán tử thành các nhóm cơ bản sau:

3.1. Các nhóm toán tử phổ biến

Nhóm Toán tử Ý nghĩa Ví dụ thực tế (với int a = 10, b = 3;)
Số học + - * / Cộng, Trừ, Nhân, Chia a / b kết quả là 3 (do đều là số nguyên)
% Chia lấy dư (Modulus) a % b kết quả là 1 (10 chia 3 dư 1)
++ -- Tăng / Giảm 1 đơn vị a++ tăng a lên 11 | b-- giảm b về 2
So sánh == != > < >= <= Bằng, Khác, Lớn, Nhỏ, Lớn hơn hoặc bằng, Nhỏ hơn hoặc bằng a != b trả về true
a == b trả về false
Logic && Phép VÀ (Vừa... vừa...) (a > 5 && b > 5) trả về false (vì b không lớn hơn 5)
|| và ! Phép HOẶC và Phép PHỦ ĐỊNH (a > 5 || b > 5) trả về true
!(a > b) trả về false
Gán = += -= *= /= %= Gán giá trị, Gán cộng dồn, Gán trừ bớt... a += 5; tương đương với a = a + 5; (a thành 15)

3.2. Code ví dụ minh họa

public class DemoOperators {
    public static void main(String[] args) {
        // 1. Bẫy phép chia số nguyên
        System.out.println(7 / 2);     // Kết quả: 3
        System.out.println(7 / 2.0);   // Kết quả: 3.5
        System.out.println(7 % 2);     // Kết quả: 1

        // 2. Toán tử logic viết chuẩn
        boolean isAdult = true;
        boolean hasTicket = false;
        System.out.println("Được vào cửa: " + (isAdult && hasTicket)); // false
        System.out.println("Được vào cửa (hoặc): " + (isAdult || hasTicket)); // true
    }
}
🚨 Lưu ý:
  • Tuyệt đối không dùng dấu khác bên toán học ($\neq$): Trong lập trình Java, dấu "Khác" bắt buộc phải viết là dấu chấm than kết hợp dấu bằng !=.
  • Phân biệt = và ==: Một dấu bằng = là phép gán (đẩy giá trị từ phải qua trái). Hai dấu bằng == mới là phép so sánh xem hai vế có bằng nhau không. Học viên làm bài tập if (a = b) là sai bản chất hoàn toàn!
  • Cơ chế Short-Circuit (Đoản mạch):
    - Với &&: Nếu vế đầu đã false, Java bỏ qua luôn không thèm tính vế sau.
    - Với ||: Nếu vế đầu đã true, Java cũng dừng lại luôn vì kết quả chắc chắn là true.

4. Điều kiện và vòng lặp

Java có nhiều cách điều khiển luồng chương trình. Dùng if cho điều kiện đơn giản, switch cho nhiều nhánh, và các vòng lặp để xử lý lặp đi lặp lại.

4.1. If / else và switch

int score = 7;
if (score >= 8) {
    System.out.println("Giỏi");
} else if (score >= 5) {
    System.out.println("Đạt");
} else {
    System.out.println("Chưa đạt");
}

int day = 3;
switch (day) {
    case 2:
        System.out.println("Thứ Hai");
        break;
    case 3:
        System.out.println("Thứ Ba");
        break;
    default:
        System.out.println("Ngày khác");
}

4.2. Vòng lặp for

for (int i = 1; i <= 10; i++) {
    System.out.print(i + " ");
}

for phù hợp khi biết trước số lần lặp.

4.3. Vòng lặp while và do while

int n = 1;
while (n <= 5) {
    System.out.println(n);
    n++;
}

int x = 1;
do {
    System.out.println(x);
    x++;
} while (x <= 3);

do while luôn chạy ít nhất một lần, dù điều kiện ban đầu sai.

4.4. Vòng lặp for each

int[] numbers = {2, 4, 6, 8};
for (int value : numbers) {
    System.out.println(value);
}

for each rất tiện khi duyệt qua mảng hoặc danh sách.

5. Quy tắc đặt tên (Naming Convention)

Trong Java, việc đặt tên thống nhất giúp code dễ đọc và chuyên nghiệp. Có 3 quy tắc viết chữ phối hợp (Case Styles) chính mà bạn bắt buộc phải nắm vững:

// 1. PascalCase (UpperCamelCase)
// 💡 Quy tắc: Viết hoa TẤT CẢ chữ cái đầu của mỗi từ.
public class StudentManager {
    // Class body
}

// 2. camelCase
// 💡 Quy tắc: Viết thường chữ đầu tiên, viết hoa từ các chữ tiếp theo.
int studentAge = 20;

public void calculateSum() {
    // Method body
}

// 3. UPPER_SNAKE_CASE
// 💡 Quy tắc: Viết hoa toàn bộ, các từ phân tách bằng dấu gạch dưới.
static final int MAX_VALUE = 100;

5.1. Khi nào sử dụng? (Bảng quy chuẩn Java)

Cộng đồng Java có quy ước bất thành văn rất nghiêm ngặt. Nhìn vào cách viết, lập trình viên có thể đoán ngay loại thành phần đó là gì:

Quy tắc Áp dụng cho Ví dụ thực tế Mẹo ghi nhớ
PascalCase Class, Interface, Enum Student, OrderService Chữ cái đầu tiên luôn "hiên ngang" viết hoa.
camelCase Biến (Variable), Hàm/Phương thức (Method) firstName, totalScore, getDetails() Giống bướu lạc đà: chữ đầu cúi xuống (viết thường), các chữ sau nhô lên (viết hoa).
UPPER_SNAKE_CASE Hằng số (Constant / final) PI, MIN_AGE, DB_URL Giống con rắn bò dài: Tất cả viết hoa và nối nhau bằng dấu gạch dưới _.

5.2. Nguyên tắc đặt tên có ý nghĩa (Clean Code)

  • Luôn đặt tên tường minh: Tránh dùng các biến vô nghĩa như a, b, c, temp trừ khi làm các biến đếm vòng lặp (như i, j).
    ❌ Sai: int d = 30; | Chuẩn: int daysInMonth = 30;
  • Động từ vs Danh từ:
    • Class/Biến nên là Danh từ hoặc cụm danh từ (Ví dụ: User, productPrice).
    • Hàm (Method) đại diện cho hành động nên phải bắt đầu bằng một Động từ (Ví dụ: save(), checkValid(), printReport()).

6. Mảng trong Java (Array)

Mảng là một tập hợp các phần tử có cùng kiểu dữ liệu, được lưu trữ liên tiếp nhau trong bộ nhớ và có kích thước cố định ngay sau khi khởi tạo.

6.1. Khai báo, khởi tạo và chỉ số (Index)

Chỉ số (Index) của mảng trong Java luôn bắt đầu từ 0 và kết thúc ở length - 1. Nếu truy cập ngoài khoảng này, chương trình sẽ báo lỗi ArrayIndexOutOfBoundsException. 

// Cách 1: Khai báo kèm khởi tạo giá trị ban đầu
int[] scores = {5, 7, 8, 9}; // Mảng có 4 phần tử (index từ 0 đến 3)

// Cách 2: Khai báo mảng trống và chỉ định kích thước trước
int[] numbers = new int[5]; // Tạo mảng chứa tối đa 5 số nguyên (mặc định ban đầu đều là 0)
numbers[0] = 10; // Gán giá trị cho phần tử đầu tiên
numbers[1] = 20;

6.2. Các thao tác cơ bản với Mảng

Dưới đây là cách truy cập, cập nhật và duyệt qua các phần tử của mảng bằng cả vòng lặp for truyền thống và vòng lặp for-each: 

int[] scores = {5, 7, 8, 9};

// 1. Truy cập phần tử qua Index
System.out.println("Phần tử đầu tiên: " + scores[0]); // Kết quả: 5

// 2. Thay đổi (Cập nhật) giá trị của một phần tử
scores[2] = 10; // Thay đổi giá trị 8 thành 10

// 3. Lấy độ dài (số lượng phần tử) của mảng
System.out.println("Độ dài của mảng: " + scores.length); // Kết quả: 4

// 4. Duyệt mảng bằng vòng lặp for (Khi cần dùng đến chỉ số i)
System.out.println("Duyệt bằng for thường:");
for (int i = 0; i < scores.length; i++) {
    System.out.println("Ví trí " + i + " có giá trị: " + scores[i]);
}

// 5. Duyệt mảng bằng vòng lặp for-each (Khi chỉ cần lấy giá trị, code ngắn gọn hơn)
System.out.println("Duyệt bằng for-each:");
for (int value : scores) {
    System.out.println(value);
}
🚨 Đặc điểm cốt lõi cần nhớ:
  • Mảng không thể co giãn (thêm/bớt phần tử) sau khi đã tạo. Muốn đổi kích thước, bắt buộc phải tạo mảng mới.
  • Khi tạo mảng bằng từ khóa new mà chưa gán giá trị, Java sẽ tự động điền các giá trị mặc định (0 với số, 0.0 với số thực, false với boolean, và null với đối tượng/String).

7. Hàm, hằng và kiểu trả về (Methods & Constants)

Để tối ưu cấu trúc mã nguồn, Java sử dụng Hàm để chia nhỏ và tái sử dụng logic, và sử dụng Hằng số để quản lý các giá trị không đổi.

7.1. Hàm và Kiểu trả về (Methods & Return Types)

Cú pháp khai báo hàm cơ bản: pham_vi_truy_cap static kieu_tra_ve ten_ham(tham_so) { ... } 

// 1. Hàm có kiểu trả về (int): Bắt buộc phải có câu lệnh return
public static int sum(int a, int b) {
    return a + b; // Trả về kết quả phép cộng
}

// 2. Hàm không có kiểu trả về (void): Thực thi hành động, không cần return
public static void sayHello(String name) {
    System.out.println("Xin chào " + name);
}

7.2. Hằng số (Constants với final)

Hằng số là các biến có giá trị chỉ được gán một lần duy nhất và không thể thay đổi trong suốt quá trình chương trình chạy.

public class DemoConfig {
    // Hằng số cấp lớp (Class-level constant): Dùng chung cho cả lớp, thường dùng kèm static
    public static final double PI = 3.14159;
    
    public static void main(String[] args) {
        // Hằng số cục bộ (Local constant): Chỉ có giá trị sử dụng bên trong hàm main
        final int MAX_SCORE = 10;
        
        // MAX_SCORE = 20; // ❌ Dòng này sẽ bị báo lỗi compile vì không thể gán lại giá trị
    }
}
🚨 Lưu ý:
  • Ý nghĩa của void: Khi một hàm không cần trả về bất kỳ giá trị tính toán nào (ví dụ chỉ in ra màn hình hoặc ghi file), bắt buộc phải sử dụng từ khóa void làm kiểu trả về.
  • Quy tắc đặt tên Hằng số: Biến khai báo với từ khóa final phải được đặt tên theo quy tắc UPPER_SNAKE_CASE (viết hoa toàn bộ, cách nhau bởi dấu gạch dưới). Điều này giúp phân biệt ngay lập tức với biến thông thường khi đọc code.
  • Phân biệt final và static final:
    - final (Cục bộ): Được khởi tạo khi hàm chứa nó được gọi.
    - static final (Toàn cục của Class): Được khởi tạo duy nhất một lần khi Class được nạp vào bộ nhớ, giúp tiết kiệm tối đa dung lượng bộ nhớ khi dùng chung một giá trị cố định.

8. String và lớp Math

String name = "Java Public";
System.out.println(name.length());
System.out.println(name.toUpperCase());
System.out.println(name.contains("Public"));

System.out.println(Math.max(10, 20));
System.out.println(Math.sqrt(16));
System.out.println(Math.pow(2, 3));

String dùng để xử lý chuỗi; Math cung cấp các hàm toán học phổ biến như max, sqrt và pow.

8.1. Bảng tổng hợp phương thức String

Phép xử lý Phương thức
Nối chuỗi +, concat()
So sánh equals(), equalsIgnoreCase()
Tìm vị trí indexOf(), lastIndexOf()
Trích xuất substring(start, end)
Thay thế replace(old, new)
Viết hoa/thường toUpperCase(), toLowerCase()

8.2. Ví dụ minh họa các phương thức String cơ bản

Hãy cùng xem cách các phương thức trên xử lý một chuỗi thực tế trong Java:

public class DemoString {
    public static void main(String[] args) {
        String str1 = "  Java Public Hè  ";
        String str2 = "java public hè";

        // 1. Xóa khoảng trắng ở hai đầu: trim()
        String cleanedStr = str1.trim();
        System.out.println("Sau khi trim: '" + cleanedStr + "'"); // "Java Public Hè"

        // 2. Lấy chiều dài chuỗi: length()
        System.out.println("Chiều dài: " + cleanedStr.length()); // 14

        // 3. Viết hoa / Viết thường: toUpperCase() / toLowerCase()
        System.out.println("Viết hoa: " + cleanedStr.toUpperCase()); // "JAVA PUBLIC HÈ"

        // 4. So sánh chuỗi (Tuyệt đối không dùng ==)
        System.out.println("So sánh bằng equals(): " + cleanedStr.equals(str2)); // false
        System.out.println("So sánh bằng ignore case: " + cleanedStr.equalsIgnoreCase(str2)); // true

        // 5. Trích xuất chuỗi con: substring(start, end) -> Lấy từ index start đến (end - 1)
        String sub = cleanedStr.substring(5, 11); 
        System.out.println("Chuỗi con (5->11): " + sub); // "Public"

        // 6. Tìm vị trí ký tự/chuỗi con: indexOf() -> Trả về -1 nếu không tìm thấy
        System.out.println("Vị trí chữ 'Public': " + cleanedStr.indexOf("Public")); // 5
        System.out.println("Ký tự tại index 0: " + cleanedStr.charAt(0)); // 'J'

        // 7. Kiểm tra chuỗi chứa / bắt đầu / kết thúc
        System.out.println("Có chứa 'Java' không? " + cleanedStr.contains("Java")); // true
        System.out.println("Có bắt đầu bằng 'Hè'? " + cleanedStr.startsWith("Hè")); // false

        // 8. Thay thế chuỗi: replace()
        String replacedStr = cleanedStr.replace("Hè", "2026");
        System.out.println("Sau khi replace: " + replacedStr); // "Java Public 2026"

        // 9. Tách chuỗi thành mảng: split()
        String[] words = cleanedStr.split(" "); // Tách nhau bởi dấu cách
        System.out.println("Số lượng từ sau khi tách: " + words.length); // 3 ("Java", "Public", "Hè")
    }
}

  3. Nhập n, tính n! và tổng số chẵn từ 1 đến n.
